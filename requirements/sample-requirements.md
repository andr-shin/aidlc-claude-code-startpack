# IT Help Desk AI 서비스 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 서비스 비전
AI 기반 자동 응답 및 티켓 분배 시스템을 통해 임직원에게는 신속하고 일관된 IT 지원 경험을, IT 운영팀에게는 효율적인 운영 환경을 제공하는 차세대 IT 서비스데스크 플랫폼

### 1.2 핵심 가치 제안

**임직원 관점**
- 24/7 즉시 응답 가능한 IT 문의 처리
- 일관되고 정확한 답변 제공

**IT 운영팀 관점**
- 1차 대응 인력 및 티켓 분배 인력 30~50% 절감
- 자동 분배를 통한 운영 효율 극대화

**시장 차별화**
- RAG 기반 사내 지식 통합으로 정확도 지속 향상
- 기존 Zendesk 시스템과 연동하는 유연한 구조

| 응답 품질 | 분산된 문서와 이력에 의존하는 지식 관리 | RAG 기반 통합으로 일관되고 정확한 답변 제공 |
| 운영 비용 | 1차 대응 및 티켓 분배에 높은 인력 투입 | AI 대체로 인력 투입 30~50% 감소 |

## 3. 주요 사용자

- **사내 임직원**: IT Help Desk를 통해 IT 문의를 제기하는 전 임직원
- **IT 운영팀 및 지원 조직**: 티켓을 처리하고 시스템을 관리하는 IT 담당자

---

## 4. 서비스 구성

### 4.1 주요 구성 요소
- **임직원용 인터페이스**: 자연어로 IT 문의를 입력하고 즉시 답변을 받는 챗봇 UI
- **관리자용 인터페이스**: IT 운영팀이 티켓 현황 및 처리 결과를 모니터링하는 관리 화면
- **AI 엔진**: RAG 기반 자동 응답 및 LLM 기반 티켓 분배 처리
- **연동 시스템**: 기존 Zendesk 시스템과의 API 연동
- **데이터 저장소**: 사내 문서, 티켓 이력, 피드백 학습 데이터 저장

## 5. 핵심 기능 요구사항

### 5.1 임직원용 기능 (Employee Features)

#### 5.1.1 자연어 기반 자동 응답 (RAG)
**목적**: 사내 데이터 기반으로 IT 문의에 즉시 답변을 생성하여 1차 해결률을 높임

**기능 요구사항**:
- 자연어 질문 입력 및 즉시 응답 생성
- 사내 문서(매뉴얼, FAQ, 정책 문서 등) 기반 RAG 검색
- 답변 근거 문서 출처 표시
- 답변 불가 시 티켓 자동 생성으로 전환
- 대화 이력 유지 (세션 내)

**성능 목표**:
- 1차 해결률 60% 이상
- 응답 생성 시간 5초 이내

**UI/UX 요구사항**:
- 챗봇 형태의 대화형 인터페이스
- 터치 및 키보드 입력 모두 지원
- 답변 유용성 피드백 버튼 (👍 / 👎)

#### 5.1.2 티켓 생성 및 상태 조회
**목적**: AI 자동 응답으로 해결되지 않은 문의를 티켓으로 전환하고 처리 현황을 확인

**기능 요구사항**:
- 미해결 문의 자동 티켓 전환
- 티켓 번호 및 접수 확인 알림
- 티켓 처리 상태 실시간 조회 (접수/진행중/완료)
- 담당자 배정 결과 확인
- Zendesk 티켓과 연동된 상태 동기화

---

### 5.2 IT 운영팀용 기능 (Admin Features)

#### 5.2.1 관리자 인증
**목적**: IT 운영팀이 관리 시스템에 안전하게 접근

**기능 요구사항**:
- 사용자명 및 비밀번호 기반 로그인
- **세션 관리**:
  - 16시간 세션 유지
  - JWT 토큰 기반 인증
  - 브라우저 새로고침 시 세션 유지
  - 16시간 후 자동 로그아웃

**보안 요구사항**:
- 비밀번호 안전한 저장 (bcrypt 해싱)
- 로그인 시도 횟수 제한
- 역할 기반 접근 제어 (관리자 / 담당자)

#### 5.2.2 LLM 기반 티켓 자동 분배 (Routing)
**목적**: 수신된 문의를 분석하여 적절한 담당 조직으로 자동 배정

**기능 요구사항**:
- LLM 기반 문의 내용 분석 및 카테고리 분류
- 담당 조직 자동 배정 (네트워크 / 보안 / 하드웨어 / 소프트웨어 등)
- 분배 결과 및 근거 표시
- 수동 재배정 기능 (담당자 직접 변경)
- 분배 이력 로그 저장

**성능 목표**:
- 분배 정확도 80% 이상
- 분배 처리 시간 3초 이내

#### 5.2.3 실시간 티켓 모니터링
**목적**: 접수된 티켓 현황을 실시간으로 확인하고 관리

**기능 요구사항**:
- **대시보드 레이아웃**:
  - 티켓 상태별 집계 (접수 / 진행중 / 완료)
  - 담당 조직별 티켓 현황
  - 미처리 티켓 강조 표시
- 티켓별 상세 정보 표시
  - 티켓 번호 및 접수 시각
  - 문의 내용 요약
  - 자동 분배 결과 및 담당자
  - 처리 상태 및 이력
- 실시간 신규 티켓 알림 (SSE 기반)
- 담당 조직 및 상태별 필터링

**업데이트 방식**:
- Server-Sent Events (SSE) 기반 실시간 통신
- 신규 티켓 2초 이내 표시

#### 5.2.4 지속적 학습 구조 관리 (Feedback Loop)
**목적**: 운영팀 처리 결과를 학습 데이터로 반영하여 AI 정확도를 점진적으로 향상

**기능 요구사항**:
- 티켓 처리 결과 피드백 입력 (답변 적절성 / 분배 정확성)
- 오답 케이스 수동 레이블링 및 정답 등록
- 학습 데이터 누적 현황 조회
- 정확도 추이 리포트 (일/주/월)
- 지식베이스 문서 추가 및 업데이트

**데이터 관리**:
- 피드백 데이터 별도 저장소 관리
- 학습 데이터 버전 관리
- 정기적 모델 재학습 트리거 설정

#### 5.2.5 Zendesk 연동 관리
**목적**: 기존 Zendesk 시스템과의 데이터 동기화 및 연동 상태 관리

**기능 요구사항**:
- Zendesk API 연동 설정 및 상태 모니터링
- 티켓 데이터 양방향 동기화
- 연동 오류 알림 및 재처리
- 동기화 이력 로그 조회

## 6. MVP 개발 범위

### 핵심 기능 (필수)
**목표**: AI 자동 응답 및 티켓 자동 분배 핵심 프로세스 구현

**임직원용**:
- 자연어 기반 자동 응답 (RAG, 사내 문서 기반)
- 답변 유용성 피드백 (👍 / 👎)
- 미해결 문의 티켓 자동 전환
- 티켓 처리 상태 조회

**IT 운영팀용**:
- 관리자 인증 (16시간 세션)
- LLM 기반 티켓 자동 분배
- 실시간 티켓 모니터링 (대시보드, SSE)
- Zendesk 연동 (티켓 생성 및 상태 동기화)
- 피드백 데이터 수집 및 저장

### 2차 개발 범위 (추후)
- 지속적 학습 구조 고도화 (자동 재학습 파이프라인)
- 정확도 추이 리포트 및 분석 도구
- 다국어 지원

## 부록

### A. 용어 정의
- **MVP**: Minimum Viable Product (최소 기능 제품)
- **RAG**: Retrieval-Augmented Generation (검색 증강 생성) — 사내 문서를 검색하여 LLM 답변 생성에 활용하는 기술
- **LLM**: Large Language Model (대규모 언어 모델)
- **SSE**: Server-Sent Events — 서버에서 클라이언트로 실시간 이벤트를 전송하는 기술
- **Zendesk**: 기존 사내 IT 티켓 관리 시스템
- **티켓 세션**: 임직원이 문의를 접수한 시점부터 IT 운영팀이 처리 완료 처리까지의 처리 단위. 완료 후 동일 문제 재문의 시 신규 티켓으로 생성.
- **1차 해결률**: 담당자 개입 없이 AI 자동 응답만으로 문의가 해결된 비율
- **분배 정확도**: LLM이 문의를 올바른 담당 조직으로 배정한 비율
