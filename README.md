# 시작하기

이 폴더에는 AI 아카데미 실습용 **AIDLC 워크플로우 규칙**이 이미 알맞은 자리에 들어가 있습니다.

## 가장 쉬운 방법 (권장) — 그냥 시작하세요

1. 평소 쓰는 AI 코딩 툴(Claude Code / Kiro / Codex 등)에서 **이 폴더를 엽니다.**
2. 이렇게 말하세요:
   > "requirements 폴더의 문서를 보고 AIDLC 워크플로우를 시작해줘."
3. 끝입니다. 나중에 코드를 만들고 실행할 때 Python/Node 가 필요해지면 **AI가 알아서 설치**합니다.
   여러분이 무언가를 설치하거나 스크립트를 더블클릭할 필요가 **없습니다.**

질문은 **한국어로, 쉬운 설명과 추천**과 함께 나옵니다. 화면(UI) 선택처럼 눈으로 봐야 하는 결정은
**목업(시안) 파일**을 만들어 보여주니, 브라우저로 열어 보고 질문 파일의 `[Answer]:` 뒤에 고른
글자(A/B/C)를 적으면 됩니다.

## 무엇이 들어 있나요?

- `requirements/` — **AIDLC가 여기서 시작하는 요구사항·제약 문서 폴더.** 아래 중 무엇이든 들어갈 수 있고,
  포털에서 고른 방식대로 채워집니다:
  - 직접 입력한 `vision.md` (+ 선택 `tech-env.md`),
  - 직접 올린 파일들 — **하나든 여러 개든** 올린 파일이 이름 그대로 모두 여기 담깁니다
    (예: `product-spec.md`, `api.md`, `constraints.md` …),
  - 또는 테이블오더 예제(`table-order-requirements.md` + `constraints.md`).
  파일 이름/개수는 자유이며, AIDLC는 이 폴더 전체를 읽고 시작합니다.
- AI 코딩 툴이 자동으로 읽는 **규칙 파일들** — 직접 신경 쓰지 않아도 됩니다.

막히면 진행자(퍼실리테이터)에게 편하게 물어보세요.
